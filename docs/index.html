Fluxo End-to-End — O Caminho do Dado
Uma leitura de temperatura do sensor até o gráfico no dashboard — passo a passo.
1
Sensor publica via MQTT
ESP32 ou Simulador publica em sensors/room/room-101/temperature com payload {"value":24.3,"timestamp":"ISO8601"}
2
Mosquitto Broker repassa
Backend subscrito via wildcard sensors/room/+/temperature recebe a mensagem com QoS 1.
3
MQTT Handler — bridge thread-safe
Callback paho-mqtt roda em thread separada. asyncio.run_coroutine_threadsafe() submete ao loop asyncio do FastAPI sem criar um loop novo — solução para o conflito com asyncpg.
4
Business Rules Engine avalia RN01–RN10
Monta RuleContext imutável com valor, presença, setpoint, override manual e horário. Cada regra é avaliada em sequência. Output: lista de RuleAction.
5
Persistência via SQLAlchemy 2.0
Insere SensorData, Command (auditoria) e Alert quando gerado. Status do device → ONLINE com last_seen_at.
6
FastAPI serve via REST
GET /api/v1/sensors/{id}/data?period=1h — resposta envelope {data, meta, links} com HATEOAS parcial (links de navegação no response body).
7
React Dashboard renderiza em tempo real
Polling a cada 10s via Axios com interceptor JWT automático. Recharts atualiza o gráfico. Zustand gerencia autenticação e persiste no localStorage.
Para que serve o Redis?
Desmistificando o componente para quem nunca viu Redis antes.
1. Blacklist de Tokens JWT — Logout Real
JWT é stateless — uma vez emitido, é válido até expirar. Sem Redis, um usuário que clicou "Logout" ainda conseguiria acessar a API com o token antigo por até 60 minutos. O Redis guarda o prefixo do token com TTL igual ao tempo restante de expiração.
2. Cache de Leituras Recentes
Endpoints consultados a cada 10 segundos pelo dashboard podem ser servidos diretamente do Redis, evitando uma query SQL a cada request. Latência sub-milissegundo garante respostas rápidas mesmo em apresentações com múltiplos acessos simultâneos.

Tópicos MQTT — Protocolo dos Sensores
Broker Mosquitto na porta 1883. Payload sempre em JSON.
DireçãoTópicoPayload de exemplo
Sensor → Brokersensors/room/{room_id}/temperature{"value":24.3,"timestamp":"ISO8601","tick":42}
Sensor → Brokersensors/room/{room_id}/humidity{"value":62.1,"timestamp":"ISO8601"}
Sensor → Brokersensors/room/{room_id}/presence{"value":1,"timestamp":"ISO8601"} (1=detectado, 0=ausência)
Backend → ACdevices/ac/{device_id}/commands{"action":"on"|"off"|"setpoint","value":23.0}
AC → Backenddevices/ac/{device_id}/feedback{"power":"on","setpoint":23.0,"timestamp":"ISO8601"}

Truque da Câmera do Celular
O LED IR emite luz infravermelha invisível ao olho humano, mas captável pela câmera frontal do celular (sem filtro IR). Para validar: abra a câmera frontal → aponte o LED → acione OFF no dashboard → se o LED piscar em roxo/lilás na tela, toda a cadeia funcionou: Dashboard → API → MQTT → ESP32 → IR. Nenhum ar-condicionado necessário.
Testando com TV ou Aparelho de Som Doméstico
Os pacotes IR de qualquer aparelho doméstico seguem o mesmo protocolo. Codifique o comando POWER OFF da sua TV com a biblioteca IRremoteESP8266 (Arduino) e aponte o LED para ela. Quando a RN01 disparar por ausência de presença, a TV apagará — comportamento rigorosamente idêntico ao de um AC real. Perfeito para demonstração em banca.
Estratégia de Rede para a ExpoTech
100% local + túnel TCP sob demanda para conectar o ESP32 físico.
Sem dependência de internet
Wi-Fi instável da feira não derruba o sistema. PostgreSQL, Redis, MQTT e API rodam no notebook.
Latência mínima
Tudo local — sem RTT de datacenter. O dashboard responde em <20ms mesmo com carga alta.
Custo zero
Nenhuma cobrança por cloud. O Ngrok free tier é suficiente para o túnel TCP durante a feira.
# 1. Adicione seu token Ngrok ao .env na raiz
NGROK_AUTHTOKEN=seu_token_aqui   # ngrok.com/dashboard

# 2. Suba (o serviço ngrok já está no docker-compose)
docker compose up -d

# 3. Descubra a URL TCP gerada
docker compose logs ngrok
# → "url":"tcp://0.tcp.sa.ngrok.io:XXXXX"
# Ou acesse: http://localhost:4040

# 4. Configure o ESP32 com esse host e porta como broker MQTT
