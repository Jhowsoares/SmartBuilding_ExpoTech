Fluxo End-to-End — O Caminho do Dado

Uma leitura de temperatura do sensor até o gráfico no dashboard.

1
Sensor publica via MQTT
ESP32 ou Simulador → tópico sensors/room/room-101/temperature
Payload: {"value": 24.3, "timestamp": "2026-05-24T23:00:00Z"}
2
Mosquitto Broker repassa
Backend subscrito via wildcard sensors/room/+/temperature — recebe a mensagem
3
MQTT Handler — bridge thread-safe
Callback paho-mqtt roda em thread separada. asyncio.run_coroutine_threadsafe() submete o processamento ao loop FastAPI sem criar um novo loop (evita conflito com asyncpg).
4
Business Rules Engine — RN01 a RN10
Monta RuleContext com valor do sensor, presença, setpoint, override manual, horário e consumo. Avalia todas as regras em sequência. Output: lista de RuleAction.
5
Persistência — PostgreSQL via SQLAlchemy
Insere SensorData, Command (auditoria) e Alert se gerado. Status do device → ONLINE.
6
REST API serve os dados
GET /api/v1/sensors/{id}/data?period=1h — resposta validada por Pydantic com envelope {data, meta, links}
7
React Dashboard renderiza
Polling a cada 10s via Axios com interceptor JWT automático. Recharts atualiza o gráfico em tempo real. Zustand gerencia autenticação e RBAC.

Para que serve o Redis?

Desmistificando o componente para quem nunca viu Redis antes.

1. Blacklist de Tokens JWT (Logout Real)
JWT é stateless — uma vez emitido, o token é válido até expirar. Sem Redis, um usuário que fez logout ainda conseguiria acessar a API com o token antigo por até 60 minutos. O Redis armazena o prefixo do token com TTL igual ao tempo restante de expiração.
2. Cache de Leituras Recentes
Endpoints consultados a cada 10 segundos pelo dashboard (como /sensors/{id}/latest) podem ser servidos diretamente do Redis, evitando uma query SQL a cada request. Em apresentações com múltiplos acessos, mantém a API responsiva.

Tópicos MQTT — Protocolo dos Sensores

Broker Mosquitto na porta 1883. Payload sempre em JSON.

DireçãoTópicoPayload
Sensor → Broker sensors/room/{room_id}/temperature {"value": 24.3, "timestamp": "ISO8601", "tick": 42}
Sensor → Broker sensors/room/{room_id}/humidity {"value": 62.1, "timestamp": "ISO8601"}
Sensor → Broker sensors/room/{room_id}/presence {"value": 1, "timestamp": "ISO8601"} (1=detectado, 0=ausência)
Backend → AC devices/ac/{device_id}/commands {"action": "on"|"off"|"setpoint", "value": 23.0}
AC → Backend devices/ac/{device_id}/feedback {"power": "on", "setpoint": 23.0, "timestamp": "ISO8601"}

Truque da Câmera do Celular
O LED IR emite luz infravermelha invisível ao olho humano, mas captável pela câmera frontal do celular (sem filtro IR). Para validar: abra a câmera frontal → aponte o LED → acione OFF no dashboard → se o LED piscar em roxo/lilás na tela, toda a cadeia funcionou: Dashboard → API → MQTT → ESP32 → IR. Nenhum ar-condicionado necessário.
Testando com TV ou Aparelho de Som Doméstico
Os pacotes IR de qualquer aparelho doméstico seguem o mesmo protocolo. Codifique o comando POWER OFF da sua TV com a biblioteca IRremoteESP8266 (Arduino) e aponte o LED para ela. Quando a RN01 disparar por ausência, a TV apagará — comportamento rigorosamente idêntico ao de um AC real.

Estratégia de Rede para a ExpoTech

100% local + túnel sob demanda para o ESP32 físico.

Por que rodar local?
Sem dependência de internet
Wi-Fi instável da feira não derruba o sistema
Latência mínima
Tudo local, sem RTT de datacenter
Custo zero
Sem cloud, sem cobrança por uso
Como conectar o ESP32 ao Docker local
# 1. Adicione seu token Ngrok ao .env na raiz do projeto
NGROK_AUTHTOKEN=seu_token_aqui   # ngrok.com/dashboard

# 2. Suba o sistema (o serviço ngrok já está no docker-compose)
docker compose up -d

# 3. Descubra a URL TCP gerada
docker compose logs ngrok
# → "url":"tcp://0.tcp.sa.ngrok.io:XXXXX"
# Ou acesse http://localhost:4040

# 4. Configure o ESP32 com essa URL e porta como broker MQTT
O Ngrok cria um túnel TCP da porta 1883 local para um endereço público. O ESP32 em qualquer rede da feira conecta a esse endereço e conversa diretamente com o Mosquitto rodando no Docker do notebook — como se estivessem na mesma rede.
